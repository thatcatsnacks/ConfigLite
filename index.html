ConfigLite: A fast, lightweight configuration management system

ConfigLite is a lightweight and minimalist configuration file (config) management system. ConfigLite is exceptionally effective at handling files that contain up to 60 settings. With an easy-to-use config syntax, and an intuitive interface for config file management, ConfigLite is perfect for projects using smaller configs.

ConfigLite Configuration File Syntax

A ConfigLite config is made up of two entities: Elements and Settings. An element is the highest-level entity, and is enclosed in braces('[' and ']'). An element name may only contain alphanumeric characters, and must be all-caps. An element is the parent object of the second entity, a setting. A setting is made up of two parts: the attribute and the value. An attribute may only contain alphanumeric characters. A value can be represented as any of Java's common data types(String, int, short, byte, boolean, char). A setting must enclose strings in double-quotes, characters in single-quotes, and all other datatypes without quotes. Finally, a setting must end with a semicolon. A config file will look similar to:

[ELEMENT_NAME]
attribute1="value1";
attribute2=2;
attribute3=true;
attribute4='c';

ConfigLite Sample Use

The ConfigLite API makes it incredibly easy to manage an existing config file, or to create a new one. To demonstrate the ease of ConfigLite, here are some sample tasks:

To add an element:

ConfigLite config = new ConfigLite();
config.addElement("ELEMENT_NAME");

To add a setting to an element:

config.addSetting("ELEMENT_NAME", "attribute_name", "value");

To add multiple settings to an element:

//Note: you can use the above method to enter multiples
config.enterElement("ELEMENT_NAME");
config.addSetting("attribute_name", "value");
config.addSetting("attribute_name2", 2);
